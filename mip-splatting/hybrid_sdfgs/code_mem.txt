10) 2026-03-12 updates (GS->SDF + SR-guided densify)
- Added `sdf_mode=gs_bootstrap` in `hybrid_sdfgs/sdf_adapter.py`:
  - builds pseudo-SDF anchors from current GS (xyz + normal from covariance axis + sigma/confidence).
  - supports periodic refresh (`--gs_bootstrap_update_interval`) to keep teacher synchronized with evolving GS.
- Extended `SDFDensifyBlock`:
  - emits SR local resamples around selected points.
  - supports SR guidance fusion in scoring via `--sdf_densify_sr_score_coef` + per-point sampled guidance.
- `train.py` now:
  - preloads per-view external prior once per iter (if provided),
  - constructs HF prior guidance map from `(prior - gt)` residual,
  - projects sampled GS points into camera view and samples guidance,
  - feeds sampled SR guidance into densify scoring,
  - keeps SR local resample regularization loss path (`--sdf_densify_sr_weight`).
- New experiment script:
  - `hybrid_sdfgs/exp_scripts/04_run_hybrid_gsbootstrap_sdfdensify_4x_lego.sh`

11) 2026-03-12 pipeline extension (explicit SR proposal anchors)
- Added explicit proposal path in `train.py`:
  1. Extract 2D SR proposal pixels from HF prior map.
  2. Backproject pixels to rays (inverse full projection).
  3. Ray sample SDF surface candidates (min |SDF| along each ray).
  4. Gate by GS/SDF consistency: `|sdf|`, GS support, GS-nearest alignment.
  5. Build local tangent-plane samples `(u,v,delta)` around accepted anchors.
  6. Merge proposal samples into SDF densify SR regularization path.
- New args namespace: `--sdf_proposal_*` (topk/thresholds/ray/gating/plane sampling/logging).
